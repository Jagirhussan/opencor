The IDASolver plugin uses IDA to solve DAEs and it can be customised through the following properties:

Maximum step The maximum step used by the solver (default: 0)

the default value of 0 means that IDA will try to use as big a step as possible. However, this setting may not work with some models. This is the case with cardiac cellular electrophysiological models that need a stimulus protocol to generate an action potential. Such a protocol is likely to be ignored by IDA, if a maximum step value of 0 is used. To address this issue, simply set the maximum step value to the length of the stimulus protocol.

Maximum number of steps The maximum number of (internal) steps used by the solver between two given points (default: 500)

the default value of 500 should be big enough for most models. However, some (very) stiff models may require this value to be increased.

Relative tolerance The relative tolerance used by the solver (default: 10-7)
Absolute tolerance The absolute tolerance used by the solver (default: 10-7)
Interpolate solution Whether the solver returns an interpolated solution (default: true)
