The PrettyCellMLView plugin can be used to edit CellML files using a text editor and a pretty CellML format. The format is compatible with the one used in COR, although it has been expanded to support:

  • Comments (both // XXX and /* XXX */).
  • The import element (i.e. CellML 1.1 compatibility).
  • The cmeta:id attribute on all CellML elements (i.e. CellML annotation support).
  • MathML: the degree qualifier for the diff element, as well as the notanumber and infinity constants.
  • An unlimited number of group types (in COR, a group can only specify one encapsulation and/or one containment type).
Limitations

The pretty CellML format has the same limitations as the COR format:

  • A piecewise statement can only be used as part of a top-level eq statement, and nested piecewise statements are not allowed.

    those are not limitations as such since an equation can always be rewritten to accommodate those 'limitations'.

    those 'limitations' make it easier to read and maintain an equation that uses a piecewise statement.

  • The reaction element is not supported (its use is discouraged and it will be removed from the forthcoming CellML 1.2 specifications).
