To get you sarted with OpenCOR, we show you how to install and run the software, and how to author and edit models. We start by creating a simple model from scratch, saving it as a CellML file and running model simulations. We next try opening existing CellML models, both from a local directory and from the CellML Model Repository.

Downloading, installing and launching OpenCOR
  • Download OpenCOR from www.opencor.ws/downloads (versions are available for Windows, Linux and OS X).
  • Install OpenCOR by running its installer (on Windows and OS X), or by uncompressing its ZIP (on Windows and OS X) or Tarball (on Linux) file.
  • Launch OpenCOR through your system (if you ran the installer) or by double clicking on [OpenCOR]/OpenCOR.vbs (or [OpenCOR]/bin/OpenCOR.exe), [OpenCOR]/bin/OpenCOR or [OpenCOR]/OpenCOR on Windows, Linux and OS X, respectively. A window will appear that looks like:

    The central area is used to interact with files. By default, no files are open, hence the OpenCOR logo is shown instead. To the sides, there are dockable windows, which provide additional features. Those windows can be dragged and dropped to the top or bottom of the central area, as shown below, or they can be individually undocked or closed. All closed windows can be re-displayed by enabling them in the View | Windows menu, or by selecting the Tools | Reset All menu. To click on Ctrl+Space (on Windows and Linux) or on Alt+Space (on OS X) removes (for less clutter) or restores those side windows.
