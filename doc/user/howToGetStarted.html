To get you sarted with OpenCOR, we show you how to install and run the software, and how to author and edit models. We start by creating a simple model from scratch, saving it as a CellML file and running model simulations. We next try opening existing CellML models, both from a local directory and from the CellML Model Repository.

Downloading, installing and launching OpenCOR
Download OpenCOR from www.opencor.ws/downloads (versions are available for Windows, Linux and OS X). Install it by running its installer (on Windows and OS X), or by uncompressing its ZIP (on Windows and OS X) or Tarball (on Linux) file. Launch OpenCOR through your system (if you ran the installer) or by double clicking on [OpenCOR]/OpenCOR.vbs (or [OpenCOR]/bin/OpenCOR.exe), [OpenCOR]/bin/OpenCOR or [OpenCOR]/OpenCOR on Windows, Linux and OS X, respectively. The central area is used to interact with files. By default, no files are open, hence the OpenCOR logo is shown instead. To the sides, there are dockable windows, which provide additional features. Those windows can be dragged and dropped to the top or bottom of the central area, as shown below, or they can be individually undocked or closed. All closed windows can be re-displayed by enabling them in the View | Windows menu, or by selecting the Tools | Reset All menu. To click on Ctrl+Space (on Windows and Linux) or on Alt+Space (on OS X) removes (for less clutter) or restores those side windows.

Creating and running a simple CellML model

In this example, we create a simple CellML model from scratch and run it. The model is the Van der Pol oscillator defined by the second order equation: $$\frac{d^2 x}{dt^2}-\mu(1-x^2)\frac{dx}{dt}+x=0$$ with initial conditions $x=-2$ and $\frac{dx}{dt}=0$. The parameter $\mu$ controls the magnitude of the damping term. To create a CellML model, we need to convert this to two first-order equations by defining the velocity $\frac{dx}{dt}$ as a new variable $y$: $$\frac{dx}{dt}=y$$ $$\frac{dy}{dt}=\mu(1-x^2)y-x$$ The initial conditions are now $x=-2$ and $y=0$.

Make sure that you are in Editing mode (check the tabs to the left of the central area) and that the CellML Text view is selected (check the tabs to the right of the central area). Next, under the File | New menu, select the CellML 1.0 File menu item and type in the following lines of code: 

def model van_der_pol_model as
    def comp main as
        var t: dimensionless {init: 0};
        var x: dimensionless {init: -2};
        var y: dimensionless {init: 0};
        var mu: dimensionless {init: 1};

        ode(x, t) = y;
        ode(y, t) = mu*(1{dimensionless}-sqr(x))*y-x;
    enddef;
enddef;

Things to note are: (i) the closing semicolon at the end of each line (apart from the first two def statements that are opening a CellML construct), (ii) the need to indicate dimensions for each variable and constant (all dimensionless in this example â€“ but more on dimensions later), (iii) the use of ode(x,t) to indicate a first-order ODE in x and t, and (iv) the use of the convenient squaring function sqr(x) for $x^2$ (a full list of mathematical operators, elements and constants supported by the CellML Text view can be found here).

Save the model using Save under the File menu (or press Ctrl+S on Windows and Linux, or Cmd+S on OS X) and choosing .cellml as the file format (.cellml is not strictly required but is best practice). With the model saved, the other editing views become available. One is the CellML Text view (the view used to enter the code above). Others are the Raw CellML view that displays the way the model is stored (note that positioning the cursor over part of the code shows the mathematics in this view also), and the Raw view that can be used to edit any text-based file.
