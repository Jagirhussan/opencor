The following tools are required in order to develop, build, test, run and package OpenCOR:

  • Git: a distributed version control system;
  • CMake 3.1 or later: a cross-platform build system;
  • Ninja (optional): a small build system with a focus on speed;
  • C++ toolchain: a set of C++ tools. The toolchain is different depending on the platform you are using:
    • Windows: Visual Studio Express 2013 with Update 4 for Windows Desktop;
    • Linux: the g++ and libgl1-mesa-dev packages in Ubuntu; and
    • OS X: Xcode 6.4 (you might also want to get PackageMaker, which can be done via the Xcode | Open Developer Tool | More Developer Tools... menu in Xcode).
  • Qt 5.4.2: this combines both Qt 5.4.2 (a cross-platform application and UI framework) and Qt Creator 3.4.1 (its supporting IDE):
    • Windows;
    • Linux 32-bit/64-bit (you might want to sudo install Qt under /opt rather than your home folder); and
    • OS X (you might want to install Qt under /Applications rather than your home folder).
  • NSIS 2.46 (optional and Windows only): a program for creating Windows installers.

Finally, you might want to update your PATH environment variable by adding the following path(s):

  • Windows: C:\Program Files\Git\bin (or C:\Program Files (x86)\Git\bin on 64-bit Windows), C:\Program Files\CMake\bin (or C:\Program Files (x86)\CMake\bin, on 64-bit Windows), C:\Qt\5.4\msvc2013\bin (or C:\Qt\5.4\msvc2013_64\bin, on 64-bit Windows) and C:\Qt\Tools\QtCreator\bin;
  • Linux: /opt/Qt/5.4/gcc/bin (or /opt/Qt/5.4/gcc_64/bin, on 64-bit Linux) in /etc/environment on Ubuntu; and
  • OS X: /Applications/Qt/5.4/clang_64/bin in /etc/profile.
