The purpose of this plugin is to give other plugins access to a simple add function, which could be used as follows: 

#include "sampleutils.h"

...

double addResult = Sample::add(3.0, 5.0);

...

In order to create such a plugin, we first need to decide on its category. Current categories are defined in the Category enum in the PluginInfo class (see [OpenCOR]/src/plugins/plugininfo.h). For this plugin, we will be using the Sample category.

To know the category of our plugin is not only needed by OpenCOR, but also to determine where the plugin's code should be put. Indeed, the code of all the plugins is located under [OpenCOR]/src/plugins/ and, more precisely, under a category folder. As you can see, there is a sample folder (see [OpenCOR]/src/plugins/sample/) and this is where our plugin's code is to be located.

The first thing we need to do is therefore to create a folder for our plugin's code. Our plugin being called Sample, we need to create a folder with that name under the sample folder, i.e. [OpenCOR]/src/plugins/sample/Sample/.
