The purpose of this plugin is to provide a test view that gives some information about the current file.

File structure 
i18n
 └─ SampleView_fr.ts
res
 └─ SampleView.qrc
src
 ├─ sampleviewplugin.cpp
 ├─ sampleviewplugin.h
 └─ sampleviewplugin.json
CMakeLists.txt
Category

Our plugin is part of the Sample category, which means that its code can be found under [OpenCOR]/src/plugins/sample/SampleView/.

Interfaces

We want our plugin to interact with OpenCOR. This means that it needs to implement some interfaces (click here for more information about interfaces).

More specifically, we want our plugin to be a view, so we need to implement both the file handling, internationalisation, plugin and view interfaces.

CMake project

As for any OpenCOR plugin, our plugin has a CMakeLists.txt file, which contents is: 

PROJECT(SampleViewPlugin)

# Add the plugin

ADD_PLUGIN(SampleView
    SOURCES
        ../../i18ninterface.cpp
        ../../plugininfo.cpp
        ../../viewinterface.cpp

        src/sampleviewplugin.cpp
        src/sampleviewwidget.cpp
    HEADERS_MOC
        src/sampleviewplugin.h
        src/sampleviewwidget.h
    UIS
        src/sampleviewwidget.ui
    INCLUDE_DIRS
        src
    PLUGINS
        Core
    QT_MODULES
        Widgets
        XmlPatterns
    QT_LIBRARIES
        QtCore
        QtGui
        QtNetwork
        QtWidgets
        QtXmlPatterns
)

Some of the interfaces our plugin implements come with a .cpp file, so we reference them (lines 7 and 8). Then, we reference the Qt modules (line 23-24) and libraries (lines 26-30) needed by our plugin.
